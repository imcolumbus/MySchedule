🛠 GitHub에서 이전 버전으로 되돌리는 방법

GitHub에 코드를 올린 후, 특정 시점의 상태로 되돌리고 싶을 때 사용할 수 있는 3가지 방법을 안내해 드립니다.

방법 1: 이전 코드 복사해서 덮어쓰기 (가장 쉽고 안전함)

복잡한 기능 없이, 과거에 잘 작동하던 코드를 그대로 가져와서 현재 파일에 다시 붙여넣는 방식입니다.

GitHub 저장소(Repository) 메인 화면으로 이동합니다.

파일 목록 우측 상단의 [숫자] commits 버튼을 클릭합니다.

되돌리고 싶은 시점(버전)의 커밋 우측에 있는 < > (Browse files) 아이콘을 누릅니다.

해당 시점의 파일 목록이 나오면, 수정할 파일(app.jsx)을 클릭합니다.

우측 상단의 [Raw] 버튼을 누르면 코드만 나옵니다. 전체 선택(Ctrl+A) 후 **복사(Ctrl+C)**합니다.

다시 현재의 main 브랜치 파일로 돌아와서 **[연필 모양 아이콘 (Edit this file)]**을 누릅니다.

기존 내용을 다 지우고 복사한 내용을 붙여넣기(Ctrl+V) 합니다.

하단의 **[Commit changes...]**를 눌러 저장하면 Vercel이 자동으로 다시 배포합니다.

방법 2: Revert(되돌리기) 기능 사용하기 (정석)

최근에 한 작업이 잘못되었을 때, 그 작업을 취소하는 '반대 커밋'을 생성하는 방식입니다.

저장소의 Commits 목록으로 들어갑니다.

취소하고 싶은 가장 최근 커밋 우측의 [...] 버튼을 누릅니다. (간혹 목록에 바로 보이지 않으면 해당 커밋 상세페이지로 들어갑니다.)

[Revert] 버튼을 클릭합니다.

"Revert '커밋메시지'"라는 새로운 커밋 창이 뜨면 [Create pull request] 또는 바로 **[Commit]**을 진행합니다.

이 방식은 기록이 남으면서 안전하게 한 단계씩 뒤로 돌아갈 때 좋습니다.

방법 3: 특정 시점으로 브랜치 새로 만들기

현재 상태는 그대로 두고, 과거 시점에서 다시 시작하고 싶을 때 사용합니다.

Commits 목록에서 원하는 지점의 **커밋 번호(7자리 영문/숫자)**를 클릭합니다.

우측 상단의 **[Browse files]**를 누릅니다.

좌측 상단의 브랜치 선택 버튼(Branch: main)을 누르고, 새로운 이름을 입력하여 **[Create branch: 이름]**을 클릭합니다.

이제 과거 시점의 코드로 구성된 새로운 브랜치가 생성됩니다.

💡 팁

Vercel 연동: GitHub에서 코드를 수정(Commit)하고 저장하면, Vercel은 이를 감지하여 약 1~2분 내에 자동으로 새로운 주소에 반영(Deploy)합니다.

파일 백업: 중요한 수정을 하기 전에는 항상 Canvas에 있는 코드를 메모장에 따로 저장해 두는 습관을 들이시면 좋습니다.
